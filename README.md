# 会議室予約システム

Googleスプレッドシートをデータベースとして使う、シンプルな会議室予約・利用状況確認システムです。
HTMLファイル2枚とGASだけで動きます。GitHub Pagesでそのまま公開できます。

---

## 画面構成

| ファイル | 内容 |
|---|---|
| `index.html` | 利用状況確認・予約フォーム・キャンセル依頼 |
| `admin.html` | 管理者画面（パスワード認証・予約削除） |

---

## しくみ

```
index.html / admin.html（GitHub Pages）
  │
  ├─ 読み取り → スプレッドシートCSV公開URL（高速）
  │
  └─ 書き込み → GAS Web API（予約登録・キャンセル依頼・削除）
                    │
                    └─ Googleスプレッドシート
```

- **読み取りは超高速**：スプレッドシートを公開CSVとして直接取得します
- **書き込みはGAS経由**：予約登録・キャンセル依頼・管理者削除のみGASに送ります
- **管理者パスワード**：スプレッドシートの設定シートで管理します

## セットアップ手順

### 1. Googleスプレッドシートを作成する

1. [Google スプレッドシート](https://sheets.google.com) で新しいスプレッドシートを作成します
2. スプレッドシートのURLを控えます（後で使います）

---

### 2. GASでスプレッドシートを初期化する

1. スプレッドシートを開きます
2. メニュー「拡張機能」→「Apps Script」を開きます
3. 次の4つのファイルを作成し、このリポジトリの内容を貼り付けます

| Apps Script側のファイル | 貼り付ける内容 |
|---|---|
| `Code.gs` | `gas/Code.gs` |
| `Api.gs` | `gas/Api.gs` |
| `Setup.gs` | `gas/Setup.gs` |
| `appsscript.json` | `gas/appsscript.json` |

> `appsscript.json`が表示されない場合は、プロジェクト設定（歯車アイコン）→「appsscript.json マニフェストをエディタで表示する」を有効にしてください

4. 関数選択で `initializeSpreadsheet` を選び、**実行**します
   - 初回は権限承認ダイアログが出ます。「許可」を選んでください
   - 実行後、スプレッドシートに「予約」「会議室」「設定」「ログ」シートが作られます

---

### 3. 会議室を設定する

スプレッドシートの「**会議室**」シートを編集します。

| カラム | 内容 |
|---|---|
| 会議室ID | 重複しない英数字（例: `room-1`） |
| 会議室名 | 画面に表示する名前（例: `第1会議室`） |
| カレンダーID | GoogleカレンダーのID（任意） |
| 利用可能 | `有効` にすると予約画面に表示されます |
| 表示順 | 数字が小さいほど上に表示されます |

> カレンダーIDを設定すると、予約時にGoogleカレンダーへ予定が登録されます。
> 設定しない場合もスプレッドシートへの予約記録は動作します。

---

### 4. 管理者パスワードを設定する

スプレッドシートの「**設定**」シートを開きます。
「管理者パスワード」行の「設定値」を変更してください。

> デフォルトは `admin1234` です。**必ず変更してください。**

---

### 5. GASをWeb APIとしてデプロイする

1. Apps Script右上の「**デプロイ**」→「**新しいデプロイ**」をクリックします
2. 種類：「**ウェブアプリ**」を選択します
3. 次のユーザーとして実行：「**自分**」
4. アクセスできるユーザー：「**全員**」
5. 「デプロイ」をクリックします
6. 表示されたWebアプリのURLを控えます（`https://script.google.com/macros/s/.../exec` の形式）

---

### 6. スプレッドシートを公開する（読み取り用）

1. スプレッドシートの「**共有**」ボタンをクリックします
2. 「**リンクを知っている全員**」に「**閲覧者**」権限を付与します

---

### 7. HTMLファイルのCONFIGを書き換える

`index.html` と `admin.html` の両方を開き、先頭付近の `const CONFIG = { ... }` を書き換えます。

```js
const CONFIG = {
  SPREADSHEET_ID: 'スプレッドシートID',   // URLの /d/【ここ】/edit
  GAS_API_URL:    'GAS Web APIのURL',       // デプロイ後のexec URL
  SHEET_GID: {
    reservations: '0',   // シートタブURLの gid=【ここ】
    rooms: '1',
    settings: '2',
  },
};
```

**スプレッドシートIDの確認方法:**
スプレッドシートのURLの `/d/` と `/edit` の間の文字列です。

**シートgidの確認方法:**
シートタブを右クリック→「このシートへのリンクを取得」でURLの `gid=` の数字です。

---

### 8. GitHub Pagesで公開する

1. GitHubにリポジトリを作成してプッシュします
2. リポジトリの「**Settings**」→「**Pages**」を開きます
3. Source：「**Deploy from a branch**」→ ブランチ `main` / フォルダ `/ (root)` を選択します
4. 「Save」をクリックします
5. しばらくするとURLが発行されます

## 各画面の使い方

### 利用状況確認（index.html）

- 日付・会議室・時間帯・会議名・団体名で絞り込みができます
- 「一覧」と「タイムライン」表示を切り替えられます
- 予約カードの「キャンセル依頼」ボタンを押すと、ステータスが「キャンセル依頼中」になりLINEへ通知が飛びます

### 予約フォーム（index.html → 予約するタブ）

1. Email・団体名・お名前を入力します
2. 利用日・開始時間・終了時間を入力すると、空き会議室が自動で表示されます
3. 会議名を入力して「確認へ進む」を押します
4. 内容を確認して「予約確定」を押します

「＋ 予約を追加」で複数の予約をまとめて登録できます。

### 管理者画面（admin.html）

1. 設定シートのパスワードを入力してログインします
2. 会議名・担当者名・団体名・会議室・ステータス・カレンダーで絞り込みができます
3. 削除したい予約にチェックを入れて「選択した予約を削除」を押します

---

## キャンセル依頼のLINE通知について

キャンセル依頼送信時に、LINEアプリが開いて送信用メッセージが自動入力されます。
メッセージにはスプレッドシートのURLが含まれるので、管理者がすぐ確認できます。

送り先はLINEで任意のトーク（個人・グループ）に手動で送信します。

## ファイル構成

```
/
├── index.html          # 予約・状況確認画面
├── admin.html          # 管理者画面
└── gas/
    ├── Code.gs         # スプレッドシート操作・予約ロジック
    ├── Api.gs          # GAS Web APIエンドポイント
    ├── Setup.gs        # スプレッドシート初期化（セットアップ時のみ使用）
    └── appsscript.json # GASマニフェスト
```

## よくあるエラー

| 症状 | 確認すること |
|---|---|
| 予約が表示されない | スプレッドシートの共有設定が「リンクを知っている全員」になっているか確認 |
| 予約できない | GAS WebアプリURLが正しいか確認。CORSエラーの場合はGASのアクセス設定を確認 |
| 管理者にログインできない | 設定シートの「管理者パスワード」の値を確認 |
| カレンダーに登録されない | 会議室シートのカレンダーIDが正しく設定されているか確認 |
| gidが違う | シートタブを右クリック→「このシートへのリンクを取得」でURLのgidを確認 |
