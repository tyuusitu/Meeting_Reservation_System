# 会議出欠管理システム 仕様書

## 1. この仕様書の目的

本仕様書は、会議の出席・欠席をメンバー本人が簡単に登録でき、管理者が会議設定・出欠状況・参加率・連続欠席状況を確認できるシステムを、AIに実装させるための指示書である。

フロントエンドは GitHub Pages で公開する静的HTMLとし、データベースは Google スプレッドシートを使用する。  
データの読み込みはフロントエンドから Google スプレッドシートを直接 `fetch` して行い、データの書き込みは GAS の WebアプリAPI 経由で行う。

スプレッドシートは非エンジニアも直接確認・編集するため、シート名・列名・設定値はすべて日本語を基本とする。

---

## 2. システム概要

### 2.1 作成する画面

本システムでは、以下の2つの画面を作成する。

1. 個人用出欠登録画面
2. 管理者画面

### 2.2 個人用出欠登録画面の役割

個人用出欠登録画面では、メンバーが自分の名前を選び、現在回答受付中の会議に対して出席または欠席を登録する。

欠席を選択した場合は、欠席理由の選択と詳細入力を必須にする。

メンバーはスプレッドシートの「メンバー一覧」に登録済みの情報から選択する。  
本人確認は行わず、簡易運用として名前選択のみで登録できるようにする。

### 2.3 管理者画面の役割

管理者画面では、簡易パスワード認証後に、以下の操作・確認ができるようにする。

- 会議の新規登録
- 会議情報の編集
- 会議の募集開始日時、募集終了日時、回答締切日時の設定
- 会議ごとの出欠状況確認
- メンバーごとの参加率確認
- 部局別参加率確認
- 欠席理由一覧の確認
- 連続欠席者の確認
- 出欠データの検索・絞り込み
- スプレッドシートへの反映確認

### 2.4 管理者パスワード

管理者画面は簡易パスワードで保護する。  
パスワードはスプレッドシートの「設定」シートで管理する。

ただし、GitHub Pages は静的ホスティングであり、完全なセキュリティを提供するものではない。  
本システムは学内団体・小規模組織向けの簡易運用を前提とする。

## 3. 技術構成

### 3.1 フロントエンド

- HTML
- JavaScript
- Tailwind CSS
- GitHub Pages

### 3.2 バックエンド

- Google Apps Script
- Google スプレッドシート

### 3.3 データ保存先

Google スプレッドシートをデータベースとして使用する。

### 3.4 読み込み方式

フロントエンドから Google スプレッドシートを直接 `fetch` して読み込む。

読み込み対象は以下。

- メンバー一覧
- 会議一覧
- 出欠回答
- 設定
- 集計シート

実装時は、Google スプレッドシートを「リンクを知っている全員が閲覧可」にし、CSV または Google Visualization API 形式で読み込む。

### 3.5 書き込み方式

フロントエンドから GAS WebアプリのURLに `POST` し、GAS側でスプレッドシートに書き込む。

書き込み対象は以下。

- 出欠回答の登録・修正
- 会議の登録・編集
- 設定変更
- 操作ログ

## 4. 重要な運用ルール

### 4.1 出欠の選択肢

出欠の選択肢は以下の2つのみとする。

- 出席
- 欠席

### 4.2 未提出者の扱い

未提出者は、集計上は「出席」として扱う。

ただし、管理画面では以下を区別して表示する。

- 出席回答済み
- 欠席回答済み
- 未提出だが出席扱い

つまり、参加率の計算では未提出を出席に含めるが、運営確認用には未提出者がわかるようにする。

### 4.3 欠席理由

欠席を選択した場合、欠席理由の選択を必須にする。

欠席理由の選択肢は以下。

- 授業
- アルバイト
- 体調不良
- 家庭の都合
- その他

また、欠席理由の詳細入力欄も用意する。

「その他」を選択した場合は、詳細入力を必須にする。  
その他以外の場合も、詳細入力欄は表示するが、必須にするかどうかは「設定」シートで変更できるようにする。

### 4.4 修正可能期間

メンバーは回答締切日時までは自分の回答を修正できる。  
回答締切日時を過ぎた場合、個人用画面からは登録・修正できない。

管理者は必要に応じて、スプレッドシート上で直接修正できる。

### 4.5 会議の表示条件

個人用出欠登録画面では、以下の条件を満たす会議のみ表示する。

- 「会議一覧」シートで有効状態が「有効」
- 現在日時が「募集開始日時」以降
- 現在日時が「募集終了日時」以前

回答できるかどうかは「回答締切日時」で判定する。

つまり、会議の表示期間と回答締切は別の項目として管理する。

## 5. 画面仕様

## 5.1 個人用出欠登録画面

### 5.1.1 ファイル名

`index.html`

### 5.1.2 目的

メンバーが自分の名前を選び、受付中の会議に対して出席・欠席を登録する。

### 5.1.3 表示内容

画面には以下を表示する。

- システム名
- 自分の名前選択欄
- 所属部局、学年、役職の自動表示
- 回答受付中の会議一覧
- 会議名
- 会議日
- 開始時間
- 終了時間
- 募集終了日時
- 回答締切日時
- 出席・欠席の選択
- 欠席理由選択
- 欠席理由詳細入力欄
- 送信ボタン
- 登録済み回答の表示
- 回答締切後の注意表示

### 5.1.4 操作の流れ

1. ユーザーが画面を開く
2. スプレッドシートからメンバー一覧を読み込む
3. スプレッドシートから会議一覧を読み込む
4. 現在募集中の会議のみ表示する
5. ユーザーが自分の名前を選ぶ
6. 選択した名前に紐づく部局、学年、役職を表示する
7. ユーザーが会議を選ぶ
8. 出席または欠席を選ぶ
9. 欠席の場合は理由を選択する
10. 必要に応じて詳細理由を入力する
11. 送信ボタンを押す
12. GAS API に送信する
13. 登録成功メッセージを表示する

### 5.1.5 入力チェック

以下の入力チェックを行う。

- 名前が選択されていない場合は送信不可
- 会議が選択されていない場合は送信不可
- 出欠が選択されていない場合は送信不可
- 欠席の場合、欠席理由が未選択なら送信不可
- 欠席理由が「その他」の場合、詳細理由が空なら送信不可
- 回答締切日時を過ぎている場合は送信不可
- 募集期間外の会議は表示しない
- 同じメンバー・同じ会議の回答がすでにある場合は新規追加ではなく更新する

### 5.1.6 スマホ対応

個人用出欠登録画面はスマホ利用を前提にする。

Tailwind CSS を使用し、以下を満たすこと。

- ボタンは押しやすいサイズにする
- 入力欄は縦並びを基本とする
- 会議一覧はカード形式で表示する
- 重要な締切情報は見やすく表示する
- エラーや成功メッセージは画面上部にわかりやすく表示する

## 5.2 管理者画面

### 5.2.1 ファイル名

`admin.html`

### 5.2.2 目的

管理者が会議の設定、出欠状況の確認、参加率の確認、連続欠席者の確認を行う。

### 5.2.3 認証

管理者画面を開くと、最初にパスワード入力欄を表示する。

パスワードが正しい場合のみ管理者画面を表示する。  
パスワードは「設定」シートの「管理者パスワード」に設定する。

認証状態はブラウザの `sessionStorage` に保存する。  
ブラウザを閉じたら再ログインが必要になるようにする。

### 5.2.4 管理者画面のメニュー

管理者画面には以下のメニューを用意する。

- ダッシュボード
- 会議管理
- 出欠一覧
- 個人別集計
- 部局別集計
- 連続欠席チェック
- 欠席理由一覧
- 設定確認

### 5.2.5 ダッシュボード

ダッシュボードには以下を表示する。

- 現在募集中の会議数
- 直近の会議
- 会議ごとの出席者数
- 会議ごとの欠席者数
- 会議ごとの未提出者数
- 全体参加率
- 部局別参加率
- 連続欠席者一覧
- 注意が必要なメンバー

### 5.2.6 会議管理

会議管理では以下ができるようにする。

- 会議の新規登録
- 会議の編集
- 会議の有効・無効切り替え
- 募集開始日時の設定
- 募集終了日時の設定
- 回答締切日時の設定

会議登録フォームには以下の項目を用意する。

- 会議ID
- 会議名
- 会議日
- 開始時間
- 終了時間
- 募集開始日時
- 募集終了日時
- 回答締切日時
- 備考
- 有効状態

会議IDは、未入力の場合はGAS側で自動採番する。  
形式は `MTG-YYYYMMDD-001` のようにする。

### 5.2.7 出欠一覧

出欠一覧では、会議ごとに以下を表示する。

- 会議名
- 会議日
- メンバー名
- 部局
- 学年
- 役職
- 回答状態
- 集計上の扱い
- 欠席理由
- 欠席理由詳細
- 回答日時
- 更新日時

表示上は、未提出者も一覧に出す。  
未提出者は回答状態を「未提出」、集計上の扱いを「出席扱い」とする。

### 5.2.8 個人別集計

個人別集計では以下を表示する。

- 名前
- 部局
- 学年
- 役職
- 対象会議数
- 出席扱い回数
- 欠席回数
- 未提出回数
- 参加率
- 連続欠席回数
- 最終回答日時

参加率の計算式は以下とする。

```text
参加率 = 出席扱い回数 ÷ 対象会議数 × 100
```

未提出は出席扱いに含める。

### 5.2.9 部局別集計

部局別集計では以下を表示する。

- 部局名
- 所属人数
- 対象会議数
- 出席扱い回数
- 欠席回数
- 未提出回数
- 参加率
- 欠席率

### 5.2.10 連続欠席チェック

連続欠席チェックでは以下を表示する。

- 名前
- 部局
- 学年
- 役職
- 現在の連続欠席回数
- 直近欠席した会議名
- 直近欠席理由
- 注意レベル

注意レベルは以下の基準とする。

- 0回〜1回：通常
- 2回：注意
- 3回以上：要確認

この基準は「設定」シートで変更できるようにする。

### 5.2.11 欠席理由一覧

欠席理由一覧では以下を表示する。

- 会議名
- 会議日
- 名前
- 部局
- 学年
- 役職
- 欠席理由
- 欠席理由詳細
- 回答日時

欠席理由で絞り込みできるようにする。

### シート名

`メンバー一覧`

### 目的

### 列定義

| 列名 | 説明 | 例 |
|---|---|---|
| メンバーID | メンバーを識別するID | MEM-001 |
| 名前 | 表示する名前 | 山田太郎 |
| 部局 | 所属部局 | 企画管理部 |
| 学年 | 学年 | 3回生 |
| 役職 | 役職 | 部長 |
| 表示順 | 名前選択欄での表示順 | 1 |
| 有効状態 | 有効または無効 | 有効 |
| 登録日時 | 登録日時 | 2026/04/27 10:00 |
| 更新日時 | 更新日時 | 2026/04/27 10:00 |

### 仕様

- 有効状態が「有効」のメンバーのみ個人用画面に表示する
- 名前が同じメンバーがいる可能性を考慮し、内部処理ではメンバーIDを使用する
- 非エンジニアが編集できるよう、ID以外は日本語でわかりやすくする

| 募集開始日時 | 個人画面に表示し始める日時 | 2026/04/20 00:00 |
| 募集終了日時 | 個人画面で表示を終了する日時 | 2026/04/30 18:00 |

- 回答締切日時を過ぎた会議は、表示されていても回答・修正不可にする
- 管理者画面から新規登録・編集できる
- スプレッドシート上で直接編集しても動作するようにする

## 6.5 出欠回答シート

### シート名

`出欠回答`

### 目的

各メンバーの出欠回答を保存する。

### 列定義

| 列名 | 説明 | 例 |
|---|---|---|
| 回答ID | 回答を識別するID | ANS-20260427-001 |
| 会議ID | 会議ID | MTG-20260427-001 |
| メンバーID | メンバーID | MEM-001 |
| 名前 | 回答時点の名前 | 山田太郎 |
| 部局 | 回答時点の部局 | 企画管理部 |
| 学年 | 回答時点の学年 | 3回生 |
| 役職 | 回答時点の役職 | 部長 |
| 出欠 | 出席または欠席 | 欠席 |
| 欠席理由 | 選択式理由 | 授業 |
| 欠席理由詳細 | 詳細理由 | 5限の授業があるため |
| 回答日時 | 初回回答日時 | 2026/04/25 12:00 |
| 更新日時 | 最終更新日時 | 2026/04/26 09:00 |
| 更新回数 | 何回更新したか | 2 |

### 仕様

- 同じ会議IDとメンバーIDの組み合わせは1件のみとする
- すでに回答がある場合は、新規行を追加せず既存行を更新する
- 欠席の場合は欠席理由を必須とする
- 欠席理由が「その他」の場合は欠席理由詳細を必須とする
- 出席の場合、欠席理由と欠席理由詳細は空欄にする

| 出席扱い回数 | 出席回答数 + 未提出数 |
| 参加率 | 出席扱い回数 ÷ 対象回答枠数 |
| 欠席率 | 欠席回答数 ÷ 対象回答枠数 |
| 最終更新日時 | 集計更新日時 |

対象回答枠数は以下とする。

```text
対象回答枠数 = 所属人数 × 対象会議数
```

## 8. 集計仕様

## 8.1 対象会議

集計対象に含める会議は以下とする。

- 有効状態が「有効」
- 会議日が現在日以前
- または管理者画面で選択された期間内の会議

初期表示では、全期間を対象にする。

---

## 8.2 未提出の扱い

未提出は出席扱いとする。

例：

対象メンバーが10人、会議に対して以下の状態の場合。

- 出席回答：6人
- 欠席回答：2人
- 未提出：2人

この場合、

```text
出席扱い人数 = 6 + 2 = 8
参加率 = 8 / 10 = 80%
```

ただし、管理画面では未提出2人がわかるようにする。

---

## 8.3 個人参加率

```text
個人参加率 = 出席扱い回数 ÷ 対象会議数 × 100
```

未提出は出席扱い回数に含める。

---

## 8.4 部局別参加率

```text
部局別参加率 = 部局内の出席扱い回数合計 ÷ 部局内の対象回答枠数 × 100
```

```text
部局内の対象回答枠数 = 部局の有効メンバー数 × 対象会議数
```

---

## 8.5 連続欠席

会議日が新しい順に確認し、欠席が連続している回数を計算する。

- 欠席：連続欠席に含める
- 出席：連続欠席を止める
- 未提出：出席扱いのため連続欠席を止める

## 9.3 スプレッドシート読み込み

読み込みは `fetch` を使用する。

実装では、シート名を指定してCSV形式で取得する関数を作成する。

例：

```javascript
async function fetchSheetCsv(sheetName) {
  const url = `https://docs.google.com/spreadsheets/d/${CONFIG.SPREADSHEET_ID}/gviz/tq?tqx=out:csv&sheet=${encodeURIComponent(sheetName)}`;
  const response = await fetch(url);
  if (!response.ok) {
    throw new Error("スプレッドシートの読み込みに失敗しました");
  }
  const csvText = await response.text();
  return parseCsv(csvText);
}
```

CSVのパース処理は、カンマや改行を含む値にも対応できる実装にする。

## 9.4 GASへの書き込み

書き込みは `fetch` で GAS Webアプリに `POST` する。

```javascript
async function postToGas(action, payload) {
  const response = await fetch(CONFIG.GAS_WEB_APP_URL, {
    method: "POST",
    body: JSON.stringify({
      action,
      payload
    })
  });

  const result = await response.json();

  if (!result.success) {
    throw new Error(result.message || "処理に失敗しました");
  }

  return result.data;
}
```

CORSの制約を避けるため、GAS側のレスポンスはJSONを返す。  
必要に応じて `ContentService.createTextOutput()` を使用する。
