# アプリ理解まとめ

## 1. アプリの概要

このリポジトリは、Google スプレッドシートをデータ保存先として使う「会議室予約」と「会議出欠管理」を統合した静的 Web アプリです。

フロントエンドは `index.html` などの静的 HTML / JavaScript / CSS で構成され、GitHub Pages などにそのまま配置できます。バックエンドは Google Apps Script で、予約登録・出欠登録・管理者操作などの書き込み処理を担当します。

データの読み込みは、フロントエンドから Google スプレッドシートの CSV を直接取得します。データの書き込みは、GAS Web アプリの API に `POST` して行います。

## 2. 技術構成

| 領域 | 内容 |
|---|---|
| フロントエンド | HTML、JavaScript、Tailwind CSS CDN、独自 CSS |
| バックエンド | Google Apps Script |
| データ保存 | Google スプレッドシート |
| 予定登録 | Google カレンダー |
| 公開想定 | GitHub Pages |
| 共通設定 | `config.js` |
| ダークモード | `dark.css`、`dark.js` |

当初の `AGENTS.md` には React Native / Expo / Supabase の記載がありますが、現在の実装は静的 Web + GAS + Google スプレッドシート構成です。

## 3. 画面構成

| ファイル | 役割 |
|---|---|
| `index.html` | トップ画面。予約、利用状況、出欠登録、管理画面への導線を表示します。 |
| `reserve.html` | 会議室予約画面。利用日・時間・会議室を指定して予約します。複数予約にも対応します。 |
| `status.html` | 会議室利用状況画面。予約状況を一覧・タイムライン・Google カレンダーで確認します。 |
| `attendance.html` | 出欠登録画面。メンバーが自分の名前を選び、受付中の会議に出席・欠席を回答します。 |
| `admin.html` | 管理画面。会議、出欠、集計、設定、予約取消を管理します。 |

## 4. 共通設定

`config.js` の `window.APP_CONFIG` が全画面の接続先を管理しています。

主な設定は次の通りです。

| 設定 | 内容 |
|---|---|
| `SPREADSHEET_ID` | 読み込み対象の Google スプレッドシート ID |
| `GAS_API_URL` | 書き込み・設定取得に使う GAS Web アプリ URL |
| `SHEET_GID` | 各シートの `gid`。設定されている場合は `export?format=csv&gid=...` を使います。 |
| `SHEET_NAME` | `gid` 未設定時のフォールバック用シート名 |
| `ABSENCE_REASONS` | 欠席理由の選択肢 |
| `CSV_CACHE_TTL_MS` | CSV 読み込み結果のブラウザ側キャッシュ時間 |

`config.js` には実際のスプレッドシート ID と GAS URL が入っています。公開リポジトリで運用する場合は、閲覧権限と個人情報の扱いに注意が必要です。

## 5. データ構造

`gas/Setup.gs` の `SHEET_NAMES` と `SHEET_HEADERS` に、作成されるシートと列定義がまとまっています。

| シート名 | 用途 |
|---|---|
| `予約` | 会議室予約データ |
| `会議室` | 会議室名、カレンダー ID、利用可否、表示順 |
| `設定` | システム名、管理者パスワード、予約時間、出欠設定など |
| `メンバー一覧` | 出欠登録対象メンバー |
| `会議一覧` | 出欠を取る会議 |
| `出欠回答` | メンバーごとの出席・欠席回答 |
| `会議別集計` | 会議単位の出欠集計 |
| `個人別集計` | メンバー単位の参加率・連続欠席 |
| `部局別集計` | 部局単位の参加率 |
| `連続欠席チェック` | 連続欠席者と注意レベル |
| `操作ログ` | 予約、出欠、管理操作、エラーの履歴 |

`setupSpreadsheet()` または `initializeSpreadsheet()` を実行すると、必要なシート、ヘッダー、初期設定、初期会議室、初期メンバー、サンプル会議が作成されます。

## 6. 会議室予約機能

`reserve.html` は、会議室を予約する利用者向け画面です。

主な流れは次の通りです。

1. GAS の `get_config` からシステム設定と有効な会議室を取得します。
2. `予約` シートの CSV を読み込み、既存予約を取得します。
3. 利用日、開始時間、終了時間を入力すると、その時間帯に空いている会議室だけを選べるようにします。
4. 利用者は団体名、お名前、会議名、会議室を入力します。
5. 確認モーダルで内容を確認します。
6. `insert_reservations` API に送信します。
7. GAS 側で重複チェック、Google カレンダー登録、スプレッドシート保存を行います。

複数予約にも対応しており、設定シートの `同時登録上限` で一度に登録できる件数を制御します。既定値は `10` 件です。

予約の重複判定は次の条件です。

```text
既存予約の開始時間 < 新規予約の終了時間
かつ
新規予約の開始時間 < 既存予約の終了時間
```

## 7. 利用状況確認機能

`status.html` は、会議室の予約状況を確認する画面です。

主な機能は次の通りです。

| 機能 | 内容 |
|---|---|
| 日付フィルター | 指定日の予約だけを表示します。 |
| 会議室フィルター | 会議室単位で絞り込みます。 |
| 時間帯フィルター | 開始時刻・終了時刻で絞り込みます。 |
| 一覧表示 | 予約をカード状に一覧表示します。 |
| タイムライン表示 | 日付・会議室ごとに時間帯の予約を横棒で表示します。 |
| カレンダー表示 | 会議室の Google カレンダーを iframe で表示します。 |

予約データは `予約` シートの CSV から読み込みます。表示対象は、主に `有効` または `キャンセル依頼` の予約です。

## 8. 出欠登録機能

`attendance.html` は、メンバー本人が会議の出欠を回答する画面です。

主な流れは次の通りです。

1. GAS の `get_public_settings` からシステム名と欠席理由設定を取得します。
2. `メンバー一覧`、`会議一覧`、`出欠回答` の CSV を読み込みます。
3. 有効なメンバーを名前選択欄に表示します。
4. 有効かつ募集期間内の会議だけを表示します。
5. メンバーが出席または欠席を選びます。
6. 欠席の場合は欠席理由が必須です。
7. 欠席理由が `その他` の場合、設定により詳細入力が必須になります。
8. `submitAttendance` API に送信します。
9. GAS 側で締切、会議、メンバー、入力値を検証して `出欠回答` シートに保存します。

同じ `会議ID` と `メンバーID` の回答がすでにある場合は、新規追加ではなく既存行を更新します。

## 9. 管理画面

`admin.html` は管理者向け画面です。最初に管理者パスワードを入力し、GAS の `verifyAdminPassword` API で検証します。認証状態とパスワードは `sessionStorage` に保存されます。

管理画面のタブは次の通りです。

| タブ | 内容 |
|---|---|
| ダッシュボード | 募集中会議、直近会議、参加率、注意者などの概要 |
| 会議管理 | 会議の新規登録・更新、集計再計算 |
| 出欠一覧 | 会議ごと・メンバーごとの回答状況。未提出も表示します。 |
| 個人別集計 | メンバーごとの参加率、欠席数、連続欠席数 |
| 部局別集計 | 部局ごとの参加率、欠席率 |
| 連続欠席 | 注意・要確認メンバーの確認 |
| 欠席理由 | 欠席理由と詳細の一覧 |
| 設定 | 設定シートの値の確認・更新 |
| 予約管理 | 会議室予約一覧、選択予約の取消更新 |

予約管理の削除操作は、物理削除ではなく予約ステータスを `取消` に更新します。GAS 側では対象の Google カレンダー予定削除も試みます。

## 10. GAS API

`gas/Api.gs` の `doGet(e)` と `doPost(e)` が Web API の入口です。

### GET API

| action | 内容 |
|---|---|
| `get_config` | 予約・利用状況画面向けの公開設定、会議室一覧、今日の日付を返します。 |
| `get_public_settings` | 出欠・管理画面向けの公開設定を返します。 |

### POST API

| action | 内容 |
|---|---|
| `insert_reservations` | 会議室予約を登録します。Google カレンダー作成とシート保存を行います。 |
| `cancel_request` | 利用者からのキャンセル依頼としてステータスを更新します。 |
| `admin_delete` | 管理者操作で予約を `取消` に更新し、カレンダー予定を削除します。 |
| `verifyAdminPassword` | 管理者パスワードを検証します。 |
| `submitAttendance` | 出欠回答を登録または更新します。 |
| `saveMeeting` | 管理者画面から会議を登録または更新します。 |
| `updateSetting` | 管理者画面から設定値を更新します。 |
| `refreshAggregations` | 出欠集計を再計算します。 |

各書き込み処理は入力検証を行い、必要に応じて `操作ログ` に成功・失敗を記録します。予約登録と出欠登録では `LockService` を使い、同時実行による競合を抑制しています。

## 11. 集計仕様

出欠集計は `refreshAttendanceAggregations_()` で再計算されます。

主な仕様は次の通りです。

| 項目 | 仕様 |
|---|---|
| 出欠の選択肢 | `出席`、`欠席` のみ |
| 未提出 | 集計上は `出席扱い` |
| 会議別集計 | 対象人数、出席回答数、欠席回答数、未提出数、参加率、欠席率 |
| 個人別集計 | 対象会議数、出席回答数、欠席回答数、未提出数、参加率、連続欠席 |
| 部局別集計 | 部局ごとの対象回答枠、参加率、欠席率 |
| 連続欠席 | 新しい会議から欠席が続く回数。出席または未提出で止まります。 |
| 注意レベル | 設定シートの `注意連続欠席回数` と `要確認連続欠席回数` で判定します。 |

## 12. 初期設定と初期データ

`gas/Setup.gs` の既定値では、主に次の設定が入ります。

| 設定キー | 既定値 |
|---|---|
| システム名 | 会議室予約・出欠管理システム |
| 管理者パスワード | admin1234 |
| その他理由詳細必須 | TRUE |
| 欠席詳細常時必須 | FALSE |
| 注意連続欠席回数 | 2 |
| 要確認連続欠席回数 | 3 |

| 利用終了時刻 | 22:00 |
| 時間刻み分 | 5 |
| 同時登録上限 | 10 |

初期会議室は次の 3 件です。

| 会議室ID | 会議室名 |
|---|---|
| `room-1` | 中執内応接室 |
| `room-2` | 階段下会議室 |
| `room-3` | 中執前会議室 |

初期メンバーは `MEM-001` から `MEM-052` までの 52 名が定義されています。サンプル会議として `MTG-20260430-001` の `4月定例会` が定義されています。

## 13. セットアップの流れ

運用開始時の大まかな流れは次の通りです。

1. Google スプレッドシートを作成します。
2. Apps Script に `gas/Setup.gs`、`gas/Api.gs`、`gas/appsscript.json` を配置します。
3. `setupSpreadsheet()` を手動実行します。
4. 必要に応じて `createRoomCalendarsAndUpdateRooms()` を実行し、会議室ごとの Google カレンダーを作成します。
5. GAS を Web アプリとしてデプロイします。
6. スプレッドシートを CSV 読み込み可能な閲覧権限にします。
7. `config.js` の `SPREADSHEET_ID`、`GAS_API_URL`、必要に応じて `SHEET_GID` を更新します。
8. 静的ファイルを GitHub Pages などで公開します。

詳細な手順は `README.md` と `docs/セットアップ手順.md` に記載されています。

## 14. セキュリティと運用上の注意

このアプリは小規模組織向けの簡易運用を前提にしています。

注意点は次の通りです。

| 注意点 | 内容 |
|---|---|
| スプレッドシート公開 | CSV 読み込みのため閲覧権限が必要です。保存する情報は最小限にしてください。 |
| 管理者パスワード | 静的フロント + 簡易パスワードのため、強固な認証ではありません。初期値 `admin1234` は必ず変更します。 |
| 本人確認 | 出欠登録は名前選択のみで、厳密な本人確認はありません。 |
| 個人情報 | 重要な個人情報や秘匿情報は保存しない運用が前提です。 |
| API URL | `config.js` に GAS URL が入るため、公開範囲に注意します。 |
| カレンダーID | 会議室シートにカレンダー ID が未設定だと予約登録に失敗します。 |

## 16. 現状理解の要点

- アプリは「会議室予約」と「会議出欠管理」の統合システムです。
- 読み込みはスプレッドシート CSV、書き込みは GAS API という分担です。
- 予約は Google カレンダーにも登録され、シートにはカレンダー予定 ID が保存されます。
- 出欠では未提出を出席扱いとして集計しますが、管理画面では未提出を区別して表示します。
- 管理画面は簡易パスワードで保護されていますが、強固な認証ではありません。
- `gas/Setup.gs` がスプレッドシート設計の中心で、`gas/Api.gs` が業務ロジックの中心です。
- 現在の実装は React Native / Supabase ではなく、静的 Web / GAS / Google スプレッドシートです。
