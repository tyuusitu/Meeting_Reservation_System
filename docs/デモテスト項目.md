## 6. 残リスク

| 内容 | 理由 | 対応 |
|---|---|---|
| Google スプレッドシート公開CSVの反映に数秒遅れが出る場合がある | Google 側の公開データ更新タイミングに依存するため | デモ時は予約後に数秒待ってから利用状況を更新する |
| Google カレンダー反映に数秒遅れが出る場合がある | Calendar API と公開 iCal の反映タイミングに依存するため | 取消後は画面更新と iCal 確認を少し待って行う |
| 会議削除は物理削除ではない | 仕様上、無効化で非表示にする設計のため | 説明時は「削除=無効化」と伝える |
